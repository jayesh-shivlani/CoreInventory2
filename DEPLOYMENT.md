# Deployment Guide (Render)

This project is deployed as a single Node web service that serves:

- Backend API (`/api/*`)
- Built frontend assets (`/` and client-side routes)

Database is PostgreSQL, configured via `DATABASE_URL`.

## 1) Push repository to GitHub

Render deploys from a Git provider. Push this repository to GitHub first.

## 2) Create a Render Blueprint

1. Open Render dashboard.
2. Select New > Blueprint.
3. Connect the GitHub repository.
4. Render detects `render.yaml` automatically.
5. Create the service.

## 3) Provision PostgreSQL and set environment variables

Set or confirm the following variables in Render:

- `DATABASE_URL` : PostgreSQL connection string (required)
- `JWT_SECRET` : secure random value (auto-generated by blueprint)
- `ALLOWED_ORIGINS` : your frontend domain, for example `https://<your-render-domain>`
- `NPM_CONFIG_PRODUCTION=false` : keeps frontend build dependencies available during build

Optional:

- `RESEND_API_KEY` and `FROM_EMAIL` for production OTP email delivery

## 4) Build and runtime behavior

`render.yaml` executes:

- backend dependency install
- frontend dependency install (including devDependencies)
- frontend production build
- backend start command

The backend serves compiled frontend files from `frontend/dist` using `FRONTEND_DIST_PATH`.

## 5) Verify deployment

Validate:

- Health endpoint: `https://<your-domain>/api/health`
- App page: `https://<your-domain>/`
- Login with demo account:
  - Email: `demo@coreinventory.app`
  - Password: `demo12345`

## Operational Notes

- Backend bootstraps schema on startup if required tables are missing.
- For production hardening, restrict CORS with `ALLOWED_ORIGINS` and rotate secrets periodically.
